# EduPlatform — фронтенд

Фронтенд для EduPlatform API: React.js + TypeScript + Tailwind CSS, React Router v6,
Context API для авторизації, Axios з interceptors для авто-оновлення JWT.

## Швидкий старт

```bash
npm install
cp .env.example .env   # за потреби зміните VITE_API_BASE_URL
npm run dev
```

Застосунок очікує бекенд на `http://localhost:3000/api/v1` (як і описано в документації
API, розділ "Налаштування середовища"). Бекенд має бути піднятий з
`CORS_ORIGIN=http://localhost:5173`, бо фронтенд звертається до нього напряму
(`withCredentials: true`), а не через dev-проксі — це потрібно, щоб httpOnly cookies
з JWT (`accessToken` / `refreshToken`) коректно передавались між origin-ами.

```bash
npm run build      # production-збірка в dist/
npm run lint        # ESLint
```

## Структура проєкту

```
src/
  api/            # Axios-шар: один файл на ресурс бекенду (courses, lessons, tests…)
  components/     # Перевикористовувані UI-примітиви (Button, Modal, CourseCard…)
  context/        # AuthContext (сесія) і ToastContext (сповіщення про помилки/успіх)
  hooks/          # useDebouncedValue тощо
  layouts/        # PublicLayout (каталог/авторизація) і DashboardLayout (кабінети)
  pages/
    auth/         # Login, Register
    public/       # Каталог курсів, сторінка курсу, публічний профіль викладача
    student/      # Мої курси, перегляд уроку, тест, профіль
    teacher/      # Мої курси, редагування курсу (деталі/уроки/тест), аналітика
    admin/        # Користувачі, модерація курсів
  routes/         # RequireAuth / RequireRole — захищені маршрути за роллю
  types/          # TypeScript-інтерфейси, що відповідають моделям бекенду
  utils/          # форматування цін/дат, витяг тексту помилки з ApiError
```

## Ключові архітектурні рішення

- **Axios + interceptors** (`src/api/client.ts`): при 401 з протермінованим
  токеном застосунок один раз тихо викликає `/auth/refresh` і повторює запит; якщо це
  не вдається — розсилається подія `auth:session-expired`, на яку підписаний
  `AuthContext`, щоб вийти з сесії на клієнті.
- **AuthContext**: оскільки токени зберігаються як httpOnly cookies (їх не видно з
  JS), при завантаженні застосунку кешований у `localStorage` об'єкт користувача
  (лише для відображення — не токен) звіряється тихим викликом `/auth/refresh`.
  Якщо сесія неактивна — стан одразу скидається.
- **Захищені маршрути**: `RequireAuth` перевіряє лише авторизацію,
  `RequireRole` — конкретну роль (студент/викладач/адмін), з редіректом на `/login`
  і збереженням `from`, щоб повернути користувача туди, звідки він прийшов.
- **Контекст для тостів**: усі помилки API показуються через `useToast().notify(...)`,
  а текст витягується з `ApiError` (`src/utils/helpers.ts → getErrorMessage`), яка
  розрізняє валідаційні помилки (`errors[]`) і загальні (`message`).

## Важливі припущення (там, де документація API неповна)

1. **`GET /categories`** — у документації описана модель `Category`, але немає
   явного переліку ендпоїнтів для довідника категорій. Припущено стандартний
   `GET /api/v1/categories` (`src/api/categories.ts`). Якщо у бекенді шлях інший —
   достатньо виправити один цей файл; фільтр категорій у каталозі і форми
   створення/редагування курсу деградують без помилок, якщо ендпоїнт відсутній
   (просто не покажуть список категорій).
2. **Відновлення сесії після перезавантаження сторінки** — у документації немає
   `GET /auth/me`. Реалізовано через кеш користувача в `localStorage` +
   валідацію через `/auth/refresh` (див. вище). Якщо бекенд згодом додасть
   `GET /auth/me`, варто замінити цю логіку в `AuthContext.tsx` на прямий запит.
3. **Поле `isEnrolled` / доступ до уроків** — для визначення, чи студент уже
   записаний на курс, фронтенд орієнтується на код відповіді `403` від
   `GET /lessons/course/:id` (немає доступу → значить не записаний).

## Дизайн

Кольорова палітра й типографіка побудовані на метафорі зошита/конспекту
("ledger"): серифний `Fraunces` для заголовків, `Inter` для основного тексту,
`IBM Plex Mono` для чисел і статусів. Прогрес у курсі/уроці показаний як
"нитка з бусинами" (`ProgressThread`) — це наскрізний візуальний елемент
платформи.
